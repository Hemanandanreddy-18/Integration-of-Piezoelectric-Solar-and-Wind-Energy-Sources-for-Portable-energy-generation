PROJECT HARDWARE SYSTEM

The application monitors a **footstep energy generation device**.

System architecture:

Footstep pressure
→ Piezoelectric sensors generate energy
→ Generated energy lights an LED
→ LDR sensor detects LED brightness
→ ESP8266 reads LDR sensor value
→ ESP8266 sends readings to Firebase
→ Mobile app visualizes and analyzes data

Important constraint:

The ESP8266 only sends data when connected to power (for example via USB to laptop).

If no new data arrives from Firebase for some time, the app must detect that the device is **offline**.

FIREBASE DATA STRUCTURE

Database path:

/sensorData

Each entry looks like:

{
"value": 30,
"timestamp_unix": 1773073937,
"timestamp_ist": "2026-03-09 22:02:17"
}

Where:

value → LDR brightness intensity (0-1023)
timestamp_unix → unix timestamp
timestamp_ist → readable timestamp

The application must **listen in real time**.

STEP DETECTION

Use threshold detection.

STEP_THRESHOLD = 18

If sensor value > 18

→ Footstep detected.

Each spike corresponds to one step.

---

ENERGY CALCULATION

Normalize sensor values.

normalized = value / 1023

Energy estimation:

energy = normalized × 5

Display energy in **Joules**.

Allow calibration factor adjustment in settings.
